Canvas

Canvas API 提供了一个通过JavaScript和HTML的 <canvas>元素来绘制图形的方式。它用于动画、游戏画面、数据可视化、图片编辑以及实时视频处理等方面。
Canvas API主要聚焦于2D图形。而同样使用 <canvas>元素的 WebGL API 则用于绘制硬件加速的2D和3D图形。
Canvas 最早有Apple引入Webkit，用于MAC OS X的 Dashboard，随后被各个浏览器实现，如今，所有的主流浏览器都支持它。

1.基本用法

canvas标签只有两个属性，即 width和height。这些都是可选属性，当没有设置宽度和高度的时候，canvas会初始化宽度为300像素和高度为150像素。该元素可以使用CSS来自定义。但是在绘制图像时会伸缩以适应它的框架尺寸，如果CSS的尺寸与初始画布的尺寸比例不一致，画布会出现扭曲。

如果你绘制出来的图像是扭曲的，尝试使用width和height属性为<canvas>明确规定宽高，而不是使用CSS

渲染上下文

canvas元素创造了一个固定大小的画布，它公开了一个或多个公开的渲染上下文，其可以用来绘制和处理要展示的内容。我们将会将注意力放在2D渲染上下文中。其他种类的上下文也许提供了不同种类的渲染方式； 比如， WebGL 使用了基于OpenGL ES的3D上下文 (“experimental-webgl”) 。

canvas起初是空白的。为了展示，首先脚本需要找到渲染上下文，然后在它的上面绘制。

canvas元素有一个叫做 getContext() 的方法，这个方法是用来获得渲染上下文和它的绘画功能。

getContext()只有一个参数，上下文的格式。对于2D图像而言，如本教程，你可以使用 CanvasRenderingContext2D。
